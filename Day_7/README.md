### 🗑️ Day 7 – SQL DELETE Statement & Aggregate Functions

Topics Covered:

DELETE Statement:
- Basic syntax: DELETE FROM table WHERE condition;
- Importance of the WHERE clause to prevent full table deletion
- Using conditions with AND, OR, IN, BETWEEN to target specific records
- Deleting data related to foreign key constraints (and cascading deletes)
- Drop vs Truncate vs delete

Aggregate Functions (Revision & Practice):
- COUNT(), SUM(), AVG(), MIN(), MAX()
- Combining aggregates with GROUP BY for summary reports
- Filtering groups with HAVING
- Ordering results of aggregate queries
- Using aliases for cleaner output
- Round(),ifnull(),coalesce()

Key Takeaways:
- Learned to delete records safely and selectively.
- Reinforced the ability to summarize, group, and filter aggregated data.
- Understood that destructive operations (DELETE) should always be paired with cautious filtering.
- Aggregates remain a core skill for reporting and analytics.
