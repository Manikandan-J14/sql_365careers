### 🧩 Sections 21 & 22 – Advanced SQL: CTEs & Temporary Tables

Topics Covered:

- Common Table Expressions (CTEs) – Section 21:  
  - Temporary result sets defined using the WITH keyword.  
  - Can be referenced within SELECT, INSERT, UPDATE, or DELETE statements.  
  - Useful for breaking complex queries into simpler, readable parts.

- Temporary Tables – Section 22:  
  - Store intermediate data temporarily during a session.  
  - Automatically dropped when the session ends.  
  - Useful for complex calculations or staging data without altering the main schema.

Key Takeaways:
- CTEs: Improve query readability and organization, especially for complex joins and aggregations.  
- Temporary Tables: Allow efficient manipulation of intermediate data without affecting the main database.  
- Both are essential for managing complex queries and large datasets effectively.
